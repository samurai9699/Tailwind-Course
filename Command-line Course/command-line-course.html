Module 1 ‚Äî Orientation

What is the terminal, why does it exist, and how does it actually work?

What is the command line?

The command line (also called the terminal, shell, or CLI) is a text-based interface to your computer. Instead of clicking icons, you type instructions and the computer responds.

The GUI (graphical interface) you use every day is just a pretty layer on top. The terminal talks directly to the operating system ‚Äî it's faster, more powerful, and universal across every machine you'll ever work on.

üìñ

Shell vs Terminal ‚Äî what's the difference?

The terminal is the window you see. The shell is the program running inside it that interprets your commands. The most common shell is bash (or zsh on modern Macs). They're almost identical for everyday use.

Anatomy of a command

Every command follows this pattern: command [options] [arguments]

bash
# Simplest form: just the command name
ls
Documents Downloads Music Pictures
# With an option (flags start with - or --)
ls -l
drwxr-xr-x 5 alex staff 160 Jan 15 09:21 Documents
# With an option AND an argument (a path)
ls -l ~/Documents
command
The action to perform ‚Äî like ls, cd, mkdir
-flag or --flag
Modifies how the command behaves. Short (-l) or long (--list)
argument
What the command acts on ‚Äî usually a file, folder, or text
spaces matter
Command parts are separated by spaces. Exact spacing required.

Create, copy, move, rename, and delete ‚Äî confidently.

Creating things

bash
# Create an empty file
touch notes.txt
(creates notes.txt if it doesn't exist, or updates its timestamp)
# Create a folder
mkdir my-project
(creates the my-project directory)
# Create nested folders in one shot (-p = make parents)
mkdir -p projects/2024/january
(creates all three folders even if projects/ doesn't exist)
# Write text directly into a file
echo "Hello, World" > notes.txt
(creates/overwrites notes.txt with that text)

Copying and Moving

bash
# Copy a file: cp source destination
cp notes.txt notes-backup.txt
(original still exists; copy created)
# Copy a whole folder (need -r for "recursive")
cp -r my-project/ my-project-backup/
(copies entire folder and its contents)
# Move (or rename): mv source destination
mv notes.txt Documents/notes.txt
(moves the file into Documents/)
# Rename a file (mv works for renaming too!)
mv old-name.txt new-name.txt
(same folder, different name = rename)

Deleting things

‚ò†Ô∏è

There is no undo

The terminal doesn't have a Recycle Bin. When you delete with rm, it's gone forever. Always double-check before pressing Enter. When in doubt, move things to a backup folder first.

bash
# Delete a file
rm notes-backup.txt
(gone forever)
# Delete a folder and everything in it
rm -rf old-project/
(-r = recursive, -f = force, no warnings)
# Safer: ask for confirmation on each file
rm -ri old-project/
(prompts you: "remove this file?" y/n)
‚ö†Ô∏è

NEVER run this command

rm -rf / ‚Äî this deletes your entire system. If anyone ever tells you to run this as a "joke" or "fix", they're trying to destroy your machine. Walk away.

Module 5 ‚Äî Pipes & Redirection

The secret power of the command line: chaining commands together.

The pipe |

The pipe | takes the output of one command and feeds it as the input of the next. This is one of the most powerful ideas in computing.

bash
# List files and pipe into less (scroll through many files)
ls -la | less
(scroll through the output)
# Search for something specific in output
ls -la | grep "txt"
-rw-r--r-- 1 alex staff 3452 notes.txt
# Count how many files are in a folder
ls | wc -l
47
# See the 5 biggest files in a folder
ls -lS | head -n 6
(sort by size, show top 5)

Redirection: saving output to files

bash
# > saves output to a file (OVERWRITES)
ls > filelist.txt
(creates/overwrites filelist.txt with ls output)
# >> APPENDS to a file (doesn't overwrite)
echo "New entry" >> log.txt
(adds to end of file)
# 2> redirects error messages to a file
ls fake-folder 2> errors.txt
(error goes to file, not screen)
# Throw away all errors (send to /dev/null = trash)
command 2> /dev/null
(errors silently discarded)

Real-world combos you'll use

bash ‚Äî pro patterns
# Find all Python files and count them
find . -name "*.py" | wc -l
23
# Search log file, show only errors, save to file
grep "ERROR" app.log | tail -n 50 > recent-errors.txt
# Sort a list and remove duplicates
cat words.txt | sort | uniq
# See processes sorted by memory usage
ps aux | sort -k4 -rn | head -10
üí°

The Unix philosophy

Each command does one thing well. Pipes let you combine simple tools into powerful workflows. This is why the command line has survived 50 years ‚Äî the same patterns work on any Unix-like system.

Module 6 ‚Äî Searching Everything

grep and find: the two commands you'll use every single day.

grep ‚Äî search inside files

grep searches for text patterns inside files. It's incredibly fast and one of the most-used commands in development.

bash
# Search for a word in a file
grep "error" app.log
2024-01-15 09:23:11 ERROR: connection refused
# Case-insensitive search (-i)
grep -i "error" app.log
(matches Error, ERROR, error, eRrOr...)
# Search recursively through all files in a folder (-r)
grep -r "TODO" ./src
./src/app.py:42: # TODO: fix this later
# Show line numbers (-n)
grep -n "def " main.py
12:def main():
45:def process_data(input):
# Show count of matches (-c)
grep -c "import" main.py
8
# Invert match: show lines that DON'T match (-v)
grep -v "#" config.txt
(filters out comment lines)

find ‚Äî search for files

find searches for files and folders by name, type, size, age, and more.

bash
# Find files by name in current directory
find . -name "config.json"
./project/config.json
# Find all .log files anywhere under /var
find /var -name "*.log"
/var/log/system.log
# Find only directories (-type d)
find . -type d -name "node_modules"
./project/node_modules
# Find files modified in the last 7 days
find . -mtime -7
(all files touched in last week)
# Find AND delete (be careful!)
find . -name "*.tmp" -delete
(deletes all .tmp files ‚Äî no undo!)

which, whereis ‚Äî find programs

bash
# Where is the python program installed?
which python3
/usr/bin/python3
# Is git installed?
which git
/usr/bin/git

Module 7 ‚Äî Permissions

Who can read, write, and run what. The basics you need for the real world.

Reading the permission string

When you run ls -l, you see a string like -rwxr-xr-- next to each file. Here's how to read it.

bash
ls -l
-rwxr-xr-- 1 alex staff 4096 Jan 15 script.sh
‚îÇ‚îî‚îÄ‚îÄ‚îò‚îî‚îÄ‚îÄ‚îò‚îî‚îÄ‚îÄ‚îò
‚îÇ ‚îÇ ‚îÇ ‚îî‚îÄ‚îÄ others (everyone else)
‚îÇ ‚îÇ ‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ group
‚îÇ ‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ owner (you)
‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ file type (- = file, d = directory)
r = read, w = write, x = execute, - = no permission

Changing permissions with chmod

bash
# Make a script executable (the most common use)
chmod +x script.sh
(adds execute permission for everyone)
# Common numeric permissions you'll see:
# 755 = owner can do all, others can read+execute
chmod 755 script.sh
(rwxr-xr-x)
# 644 = owner can read+write, others can only read
chmod 644 config.txt
(rw-r--r--)
# 600 = only owner can read+write (private files)
chmod 600 ~/.ssh/id_rsa
(rw-------) ‚Äî SSH keys must be this or SSH refuses to use them

What is a package manager?

A package manager is like an app store for the terminal. Instead of downloading installers from websites, you type one command and it handles everything ‚Äî download, install, dependencies, updates.
