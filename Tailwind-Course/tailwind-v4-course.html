Module 00 ¬∑ Introduction

Welcome to the Course

What Tailwind is, why it works, and what you'll build

What Is Tailwind CSS?

Tailwind CSS is a utility-first CSS framework. Instead of writing custom CSS in separate files, you apply small, single-purpose classes directly in your HTML. These classes map almost 1:1 to CSS properties.

Think of it like this: instead of writing .card { padding: 16px; border-radius: 8px; } in a CSS file and then using class="card", with Tailwind you just write class="p-4 rounded-lg" directly.

‚ú¶ Tailwind v4 is a Ground-Up Rewrite Version 4 (released in 2025) rewrites the core engine in Rust via a tool called Lightning CSS. It is dramatically faster, uses CSS variables natively, has zero config by default, and introduces new layout features. This course covers v4 exclusively.

Why Utility-First?

Traditional CSS makes you constantly jump between HTML and CSS files, invent class names, and end up with styles that grow uncontrollably over time. Tailwind solves this by co-locating styling with your markup ‚Äî you see the design right in your HTML. You never write a CSS file again for most projects.

Module 01 ¬∑ Setup

Installing Tailwind CSS v4

Get your project running with three different setup methods

Method 1: Vite + Tailwind (Recommended)

The fastest way to start a modern project. Vite is a lightning-fast build tool and has first-class Tailwind v4 support via an official plugin.

terminal
# 1. Create a new Vite project
npm create vite@latest my-project -- --template vanilla
cd my-project

# 2. Install Tailwind v4 and its Vite plugin
npm install tailwindcss @tailwindcss/vite

# 3. Start the dev server
npm run dev

In vite.config.js, register the plugin:

vite.config.js
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [tailwindcss()]
})

In your main CSS file (e.g. style.css), add just one line:

style.css
@import "tailwindcss";
‚ú¶ v4 Change: No tailwind.config.js needed! In Tailwind v4 there is no tailwind.config.js file by default. Configuration is done entirely in your CSS file using the @theme directive. This is a big change from v3.

Method 2: Play CDN (Zero Install, Learning Only)

Perfect for quick experiments in a plain HTML file. Not for production ‚Äî it loads the full engine in the browser.

index.html
<!DOCTYPE html>
<html>
<head>
  <script src="https://cdn.tailwindcss.com"></script>
</head>
<body>
  <h1 class="text-3xl font-bold text-blue-600">
    Hello Tailwind!
  </h1>
</body>
</html>

Method 3: React / Next.js

terminal
# Next.js automatically sets up Tailwind v4
npx create-next-app@latest my-app

# For an existing React + Vite project
npm install tailwindcss @tailwindcss/vite

Verifying It Works

Add this to any HTML element and check your browser. If the text turns big and blue, Tailwind is working:

test
<p class="text-4xl font-black text-emerald-400">
  It works! üéâ
</p>

Module 02 ¬∑ Core Concepts

How Tailwind Works

Utility classes, the scale system, arbitrary values, and the build process

The Utility-First Philosophy

Every Tailwind class does exactly one thing. p-4 means padding: 1rem. text-lg means a specific font size. rounded means border-radius: 0.25rem. You compose them to build any design.

comparison.html
<!-- ‚ùå Traditional CSS approach -->
<div class="card">Profile Card</div>

/* And somewhere in a CSS file... */
.card {
  padding: 24px;
  border-radius: 12px;
  background: white;
  box-shadow: 0 4px 12px rgba(0,0,0,.1);
}

<!-- ‚úÖ Tailwind approach ‚Äî everything visible in HTML -->
<div class="p-6 rounded-xl bg-white shadow-md">
  Profile Card
</div>

The Spacing Scale

Tailwind uses a numeric scale where each unit equals 4px (0.25rem). This creates a consistent, harmonious spacing system across your entire project.

ClassValue (rem)Value (px)
p-00rem0px
p-10.25rem4px
p-20.5rem8px
p-41rem16px
p-61.5rem24px
p-82rem32px
p-123rem48px
p-164rem64px

This scale applies to padding, margin, width, height, gap, and more ‚Äî anything spacing-related uses the same numbers. Learn it once, use it everywhere.

Arbitrary Values

Sometimes you need a value that isn't in the default scale. Tailwind v4 lets you use any value with square bracket notation:

arbitrary-values.html
<!-- Exact pixel values -->
<div class="w-[372px] h-[240px]"></div>

<!-- Custom colors -->
<div class="bg-[#1a1a2e] text-[#e94560]"></div>

<!-- CSS calc() values -->
<div class="w-[calc(100%-3rem)]"></div>

<!-- CSS custom properties -->
<div class="bg-[var(--brand-color)]"></div>

How the Build Process Works

Tailwind scans your template files for class names and only generates the CSS for classes you actually use. This means your production CSS is tiny ‚Äî typically 5‚Äì20KB ‚Äî no matter how many Tailwind classes exist.

üí° v4: Zero Config Content Detection In v4 you no longer need to tell Tailwind where your files are. The new Rust-based engine automatically detects all template files in your project. No content: [...] array needed.

Module 07 ¬∑ Responsive

Responsive Design

Mobile-first breakpoints, responsive utilities, and the container query

Mobile-First by Default

The most important rule in Tailwind: classes without a prefix apply to all screen sizes. Prefixed classes apply at that breakpoint and above. You design mobile first, then layer in larger screen styles.

PrefixMin-WidthDescription
(none)0pxAll screens ‚Äî start here
sm:640pxSmall phones landscape / tablets
md:768pxTablets
lg:1024pxLaptops
xl:1280pxDesktops
2xl:1536pxLarge monitors

<!-- Responsive text size -->

<button class="md:hidden">Hamburger</button>
‚ö†Ô∏è Common Mistake: Thinking "sm: = mobile" sm: does NOT mean "small / mobile". It means "640px and above". Your mobile styles have NO prefix. Think of breakpoints as "at this size and larger".

Module 09 ¬∑ Advanced

Tailwind v4 New Features

Everything that changed and what's brand new in 2025

The Big Changes in v4

Tailwind v4 is a near complete rewrite. Here are the key changes you must know:

‚ú¶ v4 Core Changes Summary
  • New Rust-based engine (via Lightning CSS) ‚Äî 10x faster builds
  • No tailwind.config.js by default ‚Äî config lives in CSS
  • No @tailwind directives ‚Äî replaced by @import "tailwindcss"
  • First-class CSS variables ‚Äî all theme values become CSS custom properties
  • Zero configuration required to get started
  • Native container queries included by default
  • New 3D transform utilities
  • New starting-style variant for enter animations

@theme ‚Äî CSS-First Configuration

The @theme block is where all customisation happens. It's pure CSS and generates CSS variables you can use everywhere ‚Äî in Tailwind classes, your own CSS, or JavaScript.

full-theme.css
@import "tailwindcss";

@theme {
  /* Colors */
  --color-brand: #7c6aff;
  --color-brand-50: #f0eeff;
  --color-brand-500: #7c6aff;
  --color-brand-900: #2e1f99;

  /* Fonts */
  --font-sans: 'DM Sans', sans-serif;
  --font-display: 'Syne', sans-serif;

  /* Spacing overrides */
  --spacing-18: 4.5rem;

  /* Border radius */
  --radius-card: 16px;

  /* Custom shadow */
  --shadow-glow: 0 0 40px rgba(124,106,255,.4);

  /* Breakpoints */
  --breakpoint-xs: 480px;
}

/* Use in HTML: class="bg-brand-500 font-display rounded-card shadow-glow" */
/* Use in CSS:  background: var(--color-brand-500); */

@layer ‚Äî Custom Utilities & Components

Use @layer to create reusable component classes or custom utilities that integrate naturally with Tailwind's system.
